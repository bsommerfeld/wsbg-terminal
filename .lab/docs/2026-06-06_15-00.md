# Collation — ähnliche Headlines zusammenführen statt stapeln

**Branch:** `feat/editorial-lab-per-subject` · **Status:** Backend gebaut (`HeadlineCollator`), im `.lab` verdrahtet. UI-Teil (editierbar + golden-shimmer-wipe) ist Terminal-seitig, **später**.

## Problem
Jede Einheit pusht ihre Headline unabhängig. Zwei Einheiten (oder eine Einheit über
Läufe) können **nahezu identische** Zeilen erzeugen — temporäre Dubletten, zwei Winkel
auf dasselbe Event. Stapeln wäre Lärm.

## Lösung: `HeadlineCollator` (agent-Modul, @Singleton)
- Jede frisch gepushte Headline wird **embedded** (`embeddinggemma`, lokal — **kein Yahoo**)
  und gegen die **letzten N=5 noch editierbaren** Headlines verglichen (Cosine).
- `sim ≥ COLLATE_THRESHOLD` (0.86, tunebar) → „gleiche Story, verfeinert" → die neue Zeile
  **ersetzt die alte in-place** (gleiche Row-ID), statt eine neue Row anzuhängen.
- Sonst → neue Row; Fenster gleitet (älteste fällt nach 5 raus).
- **Cross-Unit:** zwei Einheiten, die in dieselbe Zeile driften, kollabieren zu einer Row.
- **Fenster = nur die letzten 5**, weil nur die noch auf dem Schirm/editierbar sind. Eine
  weggescrollte Zeile ist „locked in" → eine spätere Dublette wird eine neue Row, nicht ein
  Rewrite der Historie.

## API
`HeadlineCollator.offer(unitId, text) → Decision(id, collated, replacedText, similarity)`.
Lab ruft es in `renderUnitDraft` nach dem Publish; bei `collated` zeigt der Trace
`↻ collated (sim 0.xx) → ersetzt frühere Zeile: "<alt>"`. `clear()` beim Reset.

## Abgrenzung zu News-Coverage (3b)
- **3b** (pro Einheit): verhindert, dass **dieselbe News** zweimal vertont wird (News-IDs).
- **Collation** (cross-unit, Headline-Text): verhindert, dass **ähnliche Headlines** doppelt
  als Rows erscheinen. Zwei verschiedene Mechanismen, ergänzen sich.

## Offen / nächste Schritte
- **Threshold 0.86 live tunen** — zu hoch = Dubletten rutschen durch; zu niedrig = verschiedene
  Stories werden fälschlich gemerged (Achtung: „nichts verschlucken" gilt auch hier — lieber
  eine Dublette als zwei echte Stories verschmolzen).
- **UI-Teil** (Terminal): letzte ~3-5 Rows editierbar, `update`-Push, golden-shimmer-wipe
  (alter Content wird weggewipt, neuer erscheint an der Stelle). Backend-Signal dafür ist die
  `Decision` mit `collated=true` + Row-ID.
- **Prod-Verdrahtung**: aktuell lab-only; bei Prod-Einzug ruft der HeadlinePublisher `offer`
  und sendet `new` vs `update(rowId)` an die Seite.
