# Contributing (Mitmachen) 💎🙌

Geil, dass du mithelfen willst! Hier ist, wie du uns helfen kannst, das **WSBG Terminal** zum Mond zu bringen, ohne dass wir alles kaputt machen.

## Wie kann ich beitragen?

### Bugs melden (Red Days) 📉
Wenn das Ding abschmiert:
*   Nutze das **Bug Report Template**.
*   Sei präzise. "Geht nicht" hilft uns nicht. "Zeigt -50% an obwohl ich eigentlich +100% haben sollte" ist ein Bug (oder ein Feature?).

### Features vorschlagen (Green Days) 📈
Wenn du eine Idee hast:
*   Nutze das **Feature Request Template**.
*   Überleg dir, ob wir das wirklich brauchen oder ob es nur Spielerei ist (Spielerei ist gut).

### Pull Requests (HODL Your Commits) 🚀
1.  **Fork** das Repo.
2.  Erstell einen **Branch** für dein Feature (`git checkout -b feature/neuer-lambo-button`).
3.  **Code** deinen Kram. Halt dich halbwegs an den Style (guck dir den vorhandenen Code an).
4.  **Teste** es! Wir haben keine QA-Abteilung, du bist die QA. Wenn es beim Starten crasht, wird der PR rejected.
5.  **Commit Messages**: Sei kreativ aber verständlich.
    *   Gut: `feat: Add rocket emoji to profit display 🚀`
    *   Schlecht: `fix stuff`
6.  **Push** und erstell einen **Pull Request**.

## Coding Style (Schönschrift für Affen) 🖍️

*   Wir nutzen Java (weil Enterprise und so).
*   Kommentare sind gut, aber der Code sollte lesbar sein.
*   Keine unnötigen Dependencies! Wir wollen keine 2GB Jar-Datei.

## Rechtliches Bla Bla

Mit deinem Beitrag erklärst du dich einverstanden, dass dein Code unter der gleichen Lizenz wie das Projekt veröffentlicht wird (Deine Verluste sind unsere Verluste 🤝). Dein Code gehört jetzt uns allen.

Danke für deine Hilfe, Affe! 🦍
