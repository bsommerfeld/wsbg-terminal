# Lang & Schwarz (ls-tc.de) — genutzte Endpoints + was die Seite sonst noch hergibt

L&S Tradecenter (`www.ls-tc.de`) ist seit dem Multi-Source-Preis-Block die
**primäre Echtzeit-EUR-Kursquelle** (Modul `lang-schwarz`, `LangSchwarzClient`,
via Browser-Joker). Undokumentierte, aber stabile `_rpc/json`-Endpoints — wie
unser Yahoo/Reddit-Ansatz. Diese Notiz hält fest, was wir nutzen UND was die
Seite darüber hinaus an schönen Daten bietet, die man später einbinden kann.

## Aktuell genutzt
- **Suche per Name** — `…/_rpc/json/.lstc/instrument/search/main?q=<name>&localeId=2`
  → `[{instrumentId, displayname, isin, wkn, categorySymbol:"STK", …}]`.
  Liefert auch die **ISIN** (die Tradegate als Fallback nutzt). Namens-
  Normalisierung nötig („Micron Technology, Inc." → „Micron").
- **Intraday-Chart** — `…/_rpc/json/instrument/chart/dataForInstrument?instrumentId=<id>&series=intraday&localeId=2`
  → `series.intraday.data` = `[ts,price]` (Spark), `info.plotlines` (Vortagskurs → Move%),
  letzter Preis. **Hat die US-Nachbörse** (deshalb primär, nicht Tradegate).

## Noch ungenutzt — Kandidaten für später
- **`series=history,flags`** im selben Chart-Endpoint → längere Historie + Event-Flags
  (Dividenden/Splits etc.) für größere Charts / 52-Wochen-Kontext.
- **Nachrichten** — `www.ls-tc.de/de/aktuelle-nachrichten` ist eine Newsseite; die
  Stock-Page (`/de/aktie/<id>`) zieht per-Instrument-News vermutlich über einen
  `_rpc/json/…`-Call (live im Netzwerkverkehr zu inspizieren). Wäre eine **deutsche
  News-Quelle für die Triangulation** (aktuell nur Yahoo + NASDAQ).
- **Stammdaten / Fundamentals** der Stock-Page (KGV, Marktkap., Dividende, Sektor) —
  könnten Tags/Tooltips an der Headline anreichern.
- **Orderbuch / Bid-Ask / Spread** — L&S ist ein Market Maker; das Orderbuch wäre
  ein Liquiditäts-/Spread-Signal.
- **Derivate auf den Basiswert** (Optionsscheine/Knock-outs/Faktor-Zertifikate) —
  L&S listet sie pro Underlying; relevant für die „wo schnell Geld machen"-Logik
  (gehebelte Spielzeuge, die der Käfig liebt). Achtung: beim Preis-Matching müssen
  diese rausgefiltert bleiben (STK-Filter), hier wären sie aber bewusst erwünscht.
- **Lightstreamer-Push** — L&S hat einen Echtzeit-Streaming-Kanal (Lightstreamer-
  Protokoll). Statt Polling könnte man Kurse live pushen. Größerer Umbau, aber die
  sauberste Echtzeit-Lösung.

## Hinweis
Alles über den geteilten `WebFetcher` (Browser-Joker → direct). `ls-tc.de` ist eine
echte Webseite → der Anker greift sauber (anders als die nackten API-Hosts
`api.nasdaq.com`/`production.dataviz.cnn.io`, die den Anchor-Override + ggf.
`credentials:'omit'` brauchen).
