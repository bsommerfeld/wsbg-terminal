# Archiv-Layer bis an die Socket-Kante — Suche, Watchlist, Markt-Stimmung (ohne UI)

Folge von `2026-06-12_03-26.md` (permanentes Headline-Archiv). Auftrag: alles
bauen bis auf die UI-Anbindung — die Seite braucht später nur noch JS-Widgets.

## Gebaut ✅

- **`HeadlineJson`** (ui/bridge): die eine HeadlineRecord→Socket-JSON-
  Serialisierung, extrahiert aus `HeadlinePublisher` (der delegiert jetzt).
  Ein Row-Shape für Live-Wire, Such-Treffer und Watchlist → ein JS-Renderer
  für alle drei.
- **`ArchiveQueryBridge`** (ui/bridge, eager): Inbound
  `{type:"archive", payload:{command, …}}`:
  - `search` (query) — Volltext über Text/Ticker/Subjects/Sektoren
  - `ticker` (symbol) — komplette Instrument-Historie (byTicker-Index)
  - `recent` (hours, default 24)
  Antwort: ein `archive-results`-Broadcast `{command, requestId?, query,
  total, items}` — items gekappt (Default 100, Max 500, newest first),
  `total` ungekappt; `requestId` wird geechot fürs Request-Matching im JS.
- **`WatchlistStore`** (database-Modul): persistierte User-Watchlist,
  `<app-data>/watchlist.json` (plain JSON-Array, write-through, Symbole
  UPPER-normalisiert, Einfüge-Reihenfolge bleibt).
- **`WatchlistBridge`** (ui/bridge, eager): Inbound `{type:"watchlist",
  {command:"add"|"remove"|"list", symbol}}` → `watchlist`-Broadcast
  `{tickers, entries:[{ticker, total, headlines:[newest 5]}]}` (Join gegen
  `HeadlineArchive.byTicker`). Auch bei Client-Open gepusht. Mehr Historie
  pro Symbol = ein `archive`/`ticker`-Query entfernt.
- **`MarketMoodPublisher`** (ui/bridge, eager): das „54% BEARISH"-Badge-
  Aggregat als `market-mood`-Broadcast `{bullish, bearish, directional,
  total, dominant, percent}` über die letzten 24h der (archiv-geseedeten)
  Wire. **Lager-Mapping:** bullish = BULLISH+FOMO+BREAKOUT+SQUEEZE (Squeeze
  = Long-Wette), bearish = BEARISH+CAPITULATION; NEUTRAL/MIXED/REVERSAL
  richtungslos → aus dem Prozent raus. percent = dominant/direktional →
  „54% BEARISH" impliziert exakt 46% bullish. Gleichstand → MIXED/50; keine
  direktionale Headline → dominant/percent null (Badge zeigt nichts
  Unehrliches). Push bei Client-Open, nach jeder Headline, 60s-Safety-Poll,
  Dedupe unveränderter Werte. Kein Reddit-Re-Read nötig — reine Zählung über
  `HeadlineRecord.sentiment`.
- AppModule: alle drei eager gebunden (hub.on-Handler müssen vor dem ersten
  Page-Load stehen). Path-Ctors von `HeadlineArchive`/`WatchlistStore` jetzt
  public (Tests + künftige Export-Tools).

## Nachtrag: Mood-Badge-UI gebaut, dunkel geschaltet ✅

Das „54% BEARISH"-Badge ist fertig implementiert und wartet nur auf den
Go-Live-Schalter (`MOOD_BADGE_LIVE = true` in `web/js/widgets/mood.js`,
eine Zeile):

- **Platzierung:** `#mood-badge` im **Reddit-Widget-Header**, direkt vor dem
  „Live"-Indikator — exakt spiegelbildlich zum EUR/USD-Badge im
  FJ-Header (beide Header tragen damit je einen Daten-Chip + Live). Begründung:
  die Stimmung IST eine Reddit-Ablesung; Titlebar wäre Chrome statt Daten,
  Footer-Slides rotieren weg.
- **Optik (User-Spec):** Pill — Hintergrund in Lagerfarbe leicht transparent
  (`color-mix` 13%), Rand hell und 100% deckend in derselben Farbe, Zahl in
  der Farbe. BEARISH=`--red`, BULLISH=`--green`, Gleichstand → „NEUTRAL" in
  `--txt-2` (theme-adaptiv). 600ms-Farb-Transition, damit ein Live-Flip
  gleitet statt springt.
- **Live-Update:** rendert auf jeden `market-mood`-Broadcast (Client-Open,
  jede Headline, 60s-Poll); ohne direktionale Headline bleibt es versteckt
  (nichts Unehrliches zeigen). Server-„MIXED" → Badge „NEUTRAL".
- Tooltip: „X bullish · Y bearish (+Z neutral)".
